  Instalación y configuración en TOMCAT SERVER
Versión en Ubuntu

  1. Descarga de Apache Tomcat:
    Visita el sitio oficial de Apache Tomcat en https://tomcat.apache.org/
    En la sección "Download" elige la versión de Tomcat que desees y descarga el archivo binario (archivo .tar.gz).
  2. Instalación en Ubuntu:

    Abre una terminal. Navega al directorio donde descargaste el archivo Tomcat y descomprime el archivo con el siguiente comando:

    tar -xvzf apache-tomcat-VERSION.tar.gz

    Mueve la carpeta de Tomcat al directorio deseado:

    sudo nano /opt/apache-tomcat-VERSION/bin/catalina.sh
  3. Configuración:

    Abre el archivo de configuración catalina.sh para definir variables de entorno:

    sudo nano /opt/apache-tomcat-VERSION/bin/catalina.sh

    Agrega la línea siguiente al principio del archivo para especificar la ruta de Java:

    export JAVA_HOME=/usr/lib/jvm/openjdk-VERSION
  4. Iniciar Apache Tomcat:

    Desde la terminal, ejecuta el siguiente comando:

    /opt/apache-tomcat-VERSION/bin/startup.sh
  5. Abre un navegador web y accede a http://localhost:8080 para el panel de administración de Tomcat. Debes poder ver la página de inicio de Tomcat.
  6. Desplegar una aplicación JAVA:

    Crea un archivo WAR (Web Application Archive) de tu aplicación Java. Puedes compilar tu aplicación con herramientas como Maven o Gradle.

    Copia el archivo WAR generado en el directorio webapps dentro del directorio de Tomcat.

    cp tu-aplicacion.war $CATALINA_HOME/webapps/

    Tomcat detectará automáticamente el archivo WAR y desplegará la aplicación.

    Accede a tu aplicación visitando http://localhost:8080/nombre-de-tu-aplicacion, donde nombre-de-tu-aplicacion es el nombre del archivo WAR sin la extensión.

    ¡Listo! Has instalado, configurado Apache Tomcat y desplegado una aplicación Java. Asegúrate de ajustar los comandos y rutas según tu entorno y configuración específica.

  Instalación y configuración en GLASSFISH SERVER
Versión en Ubuntu

  1. Descarga de Glassfish server:
    Visita el sitio oficial de GlassFish para obtener la última versión, descarga el archivo ZIP del GlassFish Server. Abre la terminal en Ubuntu y escribe los siguientes comandos:

    cd /ruta/a/tu/descarga

    unzip nombre_del_archivo.zip

    sudo mv glassfish* /opt/glassfish

  2. Configuración:
    Abre el archivo .bashrc o .zshrc en tu editor de texto favorito.
    nano ~/.bashrc
    Agrega las siguientes líneas al final del archivo:
    export GLASSFISH_HOME=/opt/glassfish
    export PATH=$PATH:$GLASSFISH_HOME/bin
    Guarda y cierra el archivo. Luego, recarga el archivo de configuración:
    source ~/.bashrc

    En la terminal, ejecuta el siguiente comando para iniciar GlassFish:
    asadmin start-domain
  3. Despliegue de una aplicación Java:
    Crea un archivo .war de tu aplicación Java (por ejemplo, mi_aplicacion.war).
    Navega al directorio autodeploy de GlassFish:
    cd $GLASSFISH_HOME/glassfish/domains/domain1/autodeploy
    Copia tu archivo .war al directorio autodeploy. GlassFish detectará automáticamente la aplicación y la desplegará.
    cp /ruta/a/tu/aplicacion.war .
  4. Acceso a la Consola de Administración
    Finalmente, abre tu navegador web, visita http://localhost:4848 e ingresa tus credenciales

  Instalación y configuración en WILDFLY SERVER
Versión en Windows

  1. Descarga de Wildfly server:

    Visita el sitio web oficial de WildFly en https://wildfly.org/downloads/.

    En la sección de descargas, busca la versión más reciente y haz clic en el enlace de descarga.

    Selecciona la distribución ZIP para Windows.

  2. Configuración:

    Abre el "Panel de control"en Windows y ve a "Sistema y seguridad" > "Sistema".

    Haz clic en "Configuración avanzada del sistema" y luego en "Variables de entorno".

    En "Variables del sistema", haz clic en "Nuevo" y agrega una nueva variable llamada JBOSS_HOME con la ruta de la carpeta donde descomprimiste WildFly (por ejemplo, C:\ruta\a\wildfly-xx.xx.xx.Final).

    Encuentra la variable Path en la sección de "Variables del sistema" y haz clic en "Editar". Agrega al final la siguiente entrada: %JBOSS_HOME%\bin.

  3. Iniciar Wildfly:

    Abre una nueva ventana de terminal (cmd) y navega a la carpeta bin dentro de la carpeta WildFly.

    Ejecuta el siguiente comando para iniciar WildFly:

    standalone.bat

  4. Desplegar una aplicación Java:

    Crea o copia tu aplicación Java (un archivo WAR) en la carpeta standalone/deployments dentro de la carpeta WildFly.

    WildFly detectará automáticamente la aplicación y la desplegará.

  5. Acceder a la aplicación:

    ¡Ya podrás acceder a tu aplicación escribiendo en tu navegador la url http://localhost:8080/tu-aplicacion e ingresando tus credenciales!

FAQ

¿Cómo desplegar una aplicación Java EE en el servidor?
Después de instalar el servidor, puedes desplegar una aplicación Java EE copiando el archivo WAR (Web Application Archive) o EAR (Enterprise Archive) en el directorio correspondiente del servidor. Alternativamente, algunos servidores proporcionan herramientas administrativas o interfaces web para facilitar el despliegue.
¿Cómo configurar un servidor web para trabajar con Java EE?
En la mayoría de los casos, los servidores de aplicaciones vienen preconfigurados para trabajar con Java EE. Sin embargo, puedes necesitar ajustar la configuración según tus necesidades. Asegúrate de configurar correctamente las variables de entorno, como JAVA_HOME, y de ajustar los archivos de configuración del servidor según sea necesario.
¿Cómo se configuran las fuentes de datos (datasources) en un servidor Java EE?
La configuración de datasources varía según el servidor. Por lo general, deberás especificar la ubicación de tu base de datos, el nombre de usuario y la contraseña en el archivo de configuración del servidor. Consulta la documentación del servidor específico para obtener detalles.
¿Cómo gestionar las dependencias de la aplicación en Java EE?
Puedes gestionar las dependencias de la aplicación utilizando herramientas como Maven o Gradle. Estas herramientas ayudan a gestionar las bibliotecas y a construir archivos de artefactos (JAR, WAR, EAR) que pueden ser fácilmente desplegados en el servidor de aplicaciones.
¿Cómo configurar y asegurar un servidor de aplicaciones Java EE?
La configuración y seguridad varían según el servidor, pero generalmente involucran la configuración de certificados SSL, ajustes de firewall, y la aplicación de prácticas de seguridad recomendadas. Consulta la documentación de seguridad del servidor específico para obtener orientación detallada.
¿Dónde puedo encontrar más recursos para aprender sobre la configuración de servidores Java EE?
Puedes encontrar información adicional en la documentación oficial de los servidores Java EE, tutoriales en línea, foros de la comunidad y libros especializados en Java EE y servidores de aplicaciones. Además, la participación en comunidades en línea puede ser útil para obtener asesoramiento de otros profesionales en el campo.
